# üåø LawnCRM Design Language Guide

This document defines the UI/UX, structural, and interaction design language to be followed by all components of the LawnCRM platform. All frontend components and layouts generated by an LLM or developer must conform to these principles.

---

## üé® Design Principles

### 1. **Clarity First**
- Prioritize readability and operational clarity over visual flash.
- Use strong visual hierarchy for headings, data tables, and action buttons.
- Data must be scannable at a glance.

### 2. **Functional Minimalism**
- Keep layouts clean and task-focused.
- Eliminate visual noise and over-design.
- Display only what is relevant to the current user and context.

### 3. **Modular UI Components with ShadCN UI**
- **All components should use [ShadCN UI](https://ui.shadcn.com)** as the base library.
- ShadCN provides accessible, headless, and Tailwind-compatible components.
- Extend components using our design tokens and Tailwind classes.
- Reuse: Cards, Tables, Forms, Tabs, Dialogs, Tooltips, Toasts, Skeletons

### 4. **Mobile, Tablet, and Desktop Responsive**
- The app must work seamlessly on iPhones, iPads, and desktop browsers.
- Prioritize route visibility and time tracking features on small screens.
- Use responsive grid layouts.
- Nav drawers collapse into hamburger menus on mobile.
- Tables should stack vertically on small screens.

### 5. **Dark Mode and Light Mode**
- Support system preference via `prefers-color-scheme` media query.
- Allow manual toggle using ShadCN theme configuration.
- Ensure all components have background, text, and border tokens for both modes.
- Tailwind colors should use CSS variables for dynamic theming:
  - `bg-primary`, `text-primary`, `border-muted`, etc.
- Maintain WCAG-compliant contrast in both themes.

## üß± UI Structure Conventions

### Forms:
- Use ShadCN `<Form>` with grouped fields and validation
- Required fields marked with an asterisk `*`
- Inline error messages appear below input
- Submit buttons disabled until valid

### Tables:
- Use ShadCN `<Table>` with sticky headers, row actions
- Include sorting icons, filter buttons, and pagination
- Tables collapse into stacked cards on mobile

### Navigation:
- Sidebar for desktop: vertical with icons and labels
- Drawer menu on mobile/tablet
- Sub-navigation via horizontal tabs (ShadCN `<Tabs>`)

## üîÑ Interaction Guidelines

- **Toasts**:
  - Use ShadCN `<Toast>` component
  - Appear top right
  - Auto-dismiss after 4‚Äì6 seconds
  - Use green for success, red for error, gray for info

- **Loading States**:
  - Use `<Skeleton>` or spinner components
  - Show overlays for full-page async operations

- **Dialogs**:
  - Must confirm destructive actions (e.g., delete)
  - Esc key and outside click should close

- **Hover & Focus**:
  - Provide visual feedback (e.g., focus ring, hover border)
  - Must be keyboard accessible

---

## üìÖ Date/Time Formatting
- Use `MM/DD/YYYY` for dates
- 12-hour format for time: `h:mm A` (e.g., 4:30 PM)
- Timestamps must include created/updated metadata where relevant

---

## üß† Smart Defaults
- Job views default to today‚Äôs date
- Forms auto-fill with most used values
- Autocomplete inputs for customers, employees, trucks, services
- Filters should persist between page reloads in same session

---

## ‚úçÔ∏è Language & Labels
- Use plain English
- Avoid jargon unless widely understood in landscaping
- Use title case for page headers (e.g., "Job Schedule")
- Use sentence case for form labels and buttons (e.g., "Assign to crew")
- Error messages should be clear, polite, and helpful

---

## üîí Security & Access UX
- Admin-only controls hidden from lower roles
- Users should only see and access what they are assigned to
- Forms and actions must enforce role-based validation on both frontend and backend

## üõ† Page Layout Standards

### üß≠ Navigation Layout

#### Desktop Sidebar:
- Fixed left sidebar (`w-60`) with logo at top
- Vertical list of nav items (icon + label)
- Sections grouped with headers (e.g., "Operations", "Resources")
- Highlight active route with background accent (`bg-[--accent]/10`)
- Collapsible on smaller desktops

#### Mobile/Tablet Drawer:
- Hamburger menu in top-left
- Slide-out drawer (`ShadCN Sheet`) with the same sidebar contents
- Closes on route change or outside click

#### Top Navbar:
- Used for mobile + small tablet layouts
- Contains:
  - Left: Hamburger icon
  - Center: Page title or app logo
  - Right: Theme toggle, user avatar menu

### üìÑ Page Shell
- Uses a consistent layout wrapper:
  - `Sidebar + main content + optional topbar`
- Page title should be `<h1 className="text-2xl font-semibold">`
- Breadcrumbs optional, placed above title if used
- Max width: `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`
- Sections should have vertical spacing `space-y-6`

### ü™ü Cards / Sections
- Use `Card` component for most content blocks
- Each section should have a heading + content
- Sections inside pages should be spaced consistently (`space-y-4` or `gap-6`)

### ‚öôÔ∏è Page Actions
- Primary actions (e.g., [+ Add], [Save]) should be top-right on desktop
- Secondary actions or filters should be top-left or below title

### üîç Search and Filters
- Include when the page lists more than 5‚Äì10 items
- Use ShadCN `<Input />` for search + `<Select />` or checkboxes for filters
- Group filters into an optional dropdown if many

### üì± Responsive Behavior
- All layouts collapse vertically on mobile
- Tables become cards or stacked lists
- Filter & action bars wrap or collapse into menu
