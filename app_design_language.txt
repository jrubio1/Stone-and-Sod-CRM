# ðŸŒ¿ LawnCRM Design Language Guide

This document defines the UI/UX, structural, and interaction design language to be followed by all components of the LawnCRM platform. All frontend components and layouts generated by an LLM or developer must conform to these principles.

---

## ðŸŽ¨ Design Principles

### 1. **Clarity First**
- Prioritize readability and operational clarity over visual flash.
- Use strong visual hierarchy for headings, data tables, and action buttons.
- Data must be scannable at a glance.

### 2. **Functional Minimalism**
- Keep layouts clean and task-focused.
- Eliminate visual noise and over-design.
- Display only what is relevant to the current user and context.

### 3. **Modular UI Components with Chakra UI**
- **All components should use [Chakra UI](https://chakra-ui.com)** as the base library.
- Chakra UI provides accessible, composable, and themeable components out of the box.
- Extend components using Chakra's theming capabilities and style props.
- Reuse: Cards, Tables, Forms, Tabs, Modals, Tooltips, Toasts, Skeletons

### 4. **Mobile, Tablet, and Desktop Responsive**
- The app must work seamlessly on iPhones, iPads, and desktop browsers.
- Prioritize route visibility and time tracking features on small screens.
- Use responsive style props (e.g., `w={{ base: '100%', md: '50%' }}`).
- Nav drawers collapse into hamburger menus on mobile.
- Tables should stack vertically on small screens.

### 5. **Dark Mode and Light Mode**
- Support system preference via Chakra UI's built-in color mode functionality.
- Allow manual toggle using a color mode switcher.
- Ensure all components have background, text, and border tokens for both modes.
- The theme should define semantic tokens for colors.

## ðŸ§± UI Structure Conventions

### Forms:
- Use Chakra UI's `<FormControl>`, `<FormLabel>`, `<Input>`, etc.
- Required fields marked with an asterisk `*`
- Inline error messages using `<FormErrorMessage>`
- Submit buttons disabled until valid

### Tables:
- Use Chakra UI's `<Table>` with `<Thead>`, `<Tbody>`, etc.
- Include sorting icons, filter buttons, and pagination
- Tables collapse into stacked cards on mobile

### Navigation:
- Sidebar for desktop: vertical with icons and labels
- Drawer menu on mobile/tablet (`<Drawer>`)
- Sub-navigation via horizontal tabs (`<Tabs>`)

|------------------|---------------------------------------------------|
| Button (Primary) | `colorScheme="green"` or `bg="accent" _hover={{ bg: "accent-hover" }}` |

| Input Field      | `bg="white" _dark={{ bg: "bg.card" }} borderColor="border.muted" color="text.primary" p={2} borderRadius="md" w="full"` |
| Table Row        | `_hover={{ bg: "gray.50" }} _dark={{ _hover: { bg: "gray.800" } }} transition="background-color 0.2s"` |
| Modal            | Use `<Modal>`, `<ModalOverlay>`, `<ModalContent>` |
| Badge (Status)   | `<Badge> with colorScheme prop`                   |

## ðŸ”„ Interaction Guidelines

- **Toasts**:
  - Use Chakra UI's `useToast` hook
  - Appear top right
  - Auto-dismiss after 4â€“6 seconds
  - Use `status` prop: `success`, `error`, `info`

- **Loading States**:
  - Use `<Skeleton>` or `<Spinner>` components
  - Show overlays for full-page async operations

- **Dialogs**:
  - Use `<AlertDialog>` for destructive actions (e.g., delete)
  - Esc key and outside click should close

- **Hover & Focus**:
  - Chakra UI provides this out of the box.
  - Must be keyboard accessible

## ðŸ›  Page Layout Standards

### ðŸ§­ Navigation Layout

#### Desktop Sidebar:
- Fixed left sidebar (`<Box w="250px">`) with logo at top
- Vertical list of nav items (`<VStack>`)
- Sections grouped with headers
- Highlight active route with `_active` style prop
- Collapsible on smaller desktops

#### Mobile/Tablet Drawer:
- Hamburger menu in top-left (`<IconButton>`)
- Slide-out drawer (`<Drawer>`) with the same sidebar contents
- Closes on route change or outside click

### ðŸ“„ Page Shell
- Uses a consistent layout wrapper:
  - `<Flex><Sidebar /><Box flex="1"><Header />{children}<Footer /></Box></Flex>`
- Page title should be `<Heading as="h1" size="xl">`
- Breadcrumbs optional, placed above title if used
- Max width: `maxW="7xl" mx="auto" px={{ base: 4, md: 6, lg: 8 }}`
- Sections should have vertical spacing `<VStack spacing={6}>`

---

This design language ensures any UI built for LawnCRM remains consistent, legible, mobile-compatible, and efficient for internal workflows. All generated interfaces must follow this structure using Chakra UI for components and styling, with light/dark mode support and full responsiveness across iPhone, iPad, and desktop.
